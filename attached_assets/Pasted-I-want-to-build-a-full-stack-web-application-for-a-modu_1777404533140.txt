I want to build a full-stack web application for a modular small business operations platform.

The system should START as a restaurant-focused platform, but must be designed to expand into retail, grocery, service businesses, and mixed-use businesses later.

Do not hardcode restaurant-only logic. Build a flexible core.

-------------------------------------
TECH STACK
-------------------------------------
- React frontend
- Node.js / Express backend
- PostgreSQL database
- Use Prisma or Drizzle ORM
- Tailwind CSS for styling
- Role-based authentication and authorization

-------------------------------------
CORE CONCEPT
-------------------------------------
This is a multi-business, multi-location platform.

Each business can:
- have multiple locations
- enable or disable modules
- manage inventory, orders, employees, and time tracking

Design everything to be modular and scalable.

-------------------------------------
MODULE SYSTEM
-------------------------------------
Create a BusinessModules system so each business can enable or disable features.

Modules:
- inventory
- orders
- employees
- scheduling
- time_tracking
- reports
- payroll_future
- recipes_future

The UI should only show enabled modules.

-------------------------------------
AUTHENTICATION & ROLES
-------------------------------------
Users can:
- sign up
- log in
- create a business

Roles:
- Admin/Owner:
  full access to everything

- Manager:
  manage inventory, orders, employees, scheduling
  approve time entries
  view reports

- Cashier:
  create orders
  view items
  basic customer lookup

- HR/Payroll (future-ready):
  view employees
  view approved time entries
